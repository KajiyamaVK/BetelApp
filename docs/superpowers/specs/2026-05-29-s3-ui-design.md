# s3-ui — Design Spec

**Date:** 2026-05-29  
**Status:** Approved  
**Location:** `src/s3-ui/`

## Overview

A web dashboard for managing BetelApp lesson content stored in MinIO (S3-compatible). Users upload, preview, and soft-delete audio (MP3) and PDF files per lesson. Administrators can also manage user accounts.

---

## Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 14 (App Router) |
| Database ORM | Prisma |
| Database | PostgreSQL (homelab Docker container `postgres-postgis`) |
| Validation | Zod |
| Styling | Tailwind CSS + Shadcn/ui |
| Font | Poppins (matches mobile app) |
| Auth | JWT via httpOnly cookie |
| Storage | MinIO (`minio-client` SDK) |
| Unit/Integration tests | Jest + React Testing Library |
| E2E tests | Playwright |
| Infra | Docker (homelab) |

## Data Model (Prisma)

```prisma
model User {
  id           Int      @id @default(autoincrement())
  username     String   @unique
  passwordHash String
  isAdmin      Boolean  @default(false)
  createdAt    DateTime @default(now())
}

model Lesson {
  id    Int    @id
  title String
}
```

**Responsibilities:**
- `User` — authentication and authorization only.
- `Lesson` — stores editable titles (1–24). The source of truth for file paths, checksums, and version history remains `manifest.json` in MinIO.

---

## Authentication

- Login issues a signed JWT (`HS256`) stored as an httpOnly cookie (`token`, 7-day expiry).
- `middleware.ts` validates the cookie on every request to `/(dashboard)/*`.
- Admin-only routes (`/users`, `POST /api/users`) additionally check `payload.isAdmin`.
- Passwords are hashed with `bcrypt` (cost factor 12).

---

## Manifest Integration

The `manifest.json` in MinIO is the source of truth for file state. The dashboard reads it on every lesson list load and writes it on every upload or soft-delete.

**Soft delete behavior:** removes the `active` path from the manifest entry (sets it to `null`) and appends the old path to `history[]`. The S3 file is not deleted.

**Upload behavior:** uploads the new file to MinIO as `audio_vN.mp3` or `lesson_vN.pdf` (incrementing `N`), moves the previous active path to `history[]`, and sets the new path as `active`.

## UI / UX

### Navigation

- **Desktop:** fixed 240px sidebar. Items: "Lições" (all users), "Usuários" (admin only). Footer: username + logout button.
- **Mobile:** sidebar becomes a `<Drawer>` (Shadcn/ui), opened by a hamburger button in the top header.

### Lessons Page (`/lessons`)

- Vertical accordion list of 24 lessons.
- **Collapsed:** lesson number + title + status badges (🎵 ✓/⚠, 📄 ✓/⚠).
- **Expanded:**
  - **Title:** clicking activates an inline `<input>`. Enter or blur saves via `PUT /api/lessons/[id]`.
  - **Audio row:** file present → mini-player (play/pause circle button + progress bar + timestamp) + soft delete button. File absent → Upload button.
  - **PDF row:** file present → underlined filename link (triggers viewer) + soft delete button. File absent → Upload button.
- Delete always opens an `<AlertDialog>` confirmation before executing.

### PDF Viewer

- **Desktop:** clicking the PDF filename opens a fixed-width split panel (50% of content area) to the right of the list. Closes with ✕ in the panel header. "Open in new tab ↗" link at the bottom.
- **Mobile:** opens fullscreen overlay. Floating ✕ button fixed to the top-right corner.

### Mini-Player (audio)

- Inline within the lesson row.
- Controls: circular play/pause button (`#FFBD00` background) + progress bar + `MM:SS / MM:SS` timestamp.
- No volume control, no seek (click on progress bar is a nice-to-have, not required).

### Users Page (`/users`)

- Accessible only to admins. Non-admins redirected to `/lessons`.
- Create form at the top: username input + password input + "Administrador" checkbox + "Criar" button.
- Table below: username, admin badge, created date. No edit, no delete for now.

### Login Page

- Centered card on `#F5F5F5` background. BetelApp logo at top.
- Fields: username + password. "Entrar" button (`#FFBD00`).
- Error message below button on invalid credentials.

## Testing Strategy

### Unit Tests (Jest)

| File | What's tested |
|---|---|
| `lib/auth.ts` | JWT sign, verify, expiry, invalid token |
| `lib/manifest.ts` | Read manifest, soft delete (null active + history append), upload (version bump) |
| Zod schemas | Login, create user, upload — valid and invalid inputs |

### Integration Tests (Jest + test database)

Uses `betelapp_dev` and `betelapp-content-dev`. Each test suite resets relevant DB rows and manifest state in `beforeEach`.

| Route | Scenarios |
|---|---|
| `POST /api/auth/login` | Valid credentials, wrong password, unknown user |
| `GET /api/auth/me` | Valid token, expired token, no token |
| `GET /api/lessons` | Returns 24 lessons with manifest state merged |
| `PUT /api/lessons/[id]` | Updates title, requires auth, rejects empty title |
| `POST /api/lessons/[id]/upload` | Uploads file, bumps version, updates manifest |
| `DELETE /api/lessons/[id]/file` | Soft deletes, keeps S3 file, requires auth |
| `POST /api/users` | Admin creates user, non-admin gets 403 |

### Component Tests (React Testing Library)

| Component | What's tested |
|---|---|
| `MiniPlayer` | Renders play button, toggles to pause on click, displays timestamp |
| `LessonRow` | Shows upload button when file absent, shows player/link when present |
| `FileRow` | Delete button triggers confirmation dialog |
| `CreateUserForm` | Submits correct payload, shows error on duplicate username |

### E2E Tests (Playwright)

Runs against the dev environment (`betelapp_dev` + `betelapp-content-dev`). Seed runs before the suite.

| Flow | Steps |
|---|---|
| Login valid | Fill credentials → redirected to `/lessons` |
| Login invalid | Fill wrong password → error message visible |
| Upload audio | Expand lesson → upload MP3 → mini-player appears |
| Upload PDF | Expand lesson → upload PDF → filename link appears |
| Delete file | Click delete → confirm dialog → file row shows upload button |
| Edit title | Click title → type new name → Enter → title updated in list |
| Admin: access users | Login as admin → navigate to Usuários → page loads |
| Non-admin: block users | Login as non-admin → navigate to `/users` → redirected to `/lessons` |

## Infrastructure

### Dockerfile (multi-stage)

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY . .
RUN npm ci && npm run build

FROM node:20-alpine AS runner
WORKDIR /app
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public
ENV NODE_ENV=production
EXPOSE 3000
CMD ["node", "server.js"]
```

### Seed

`prisma/seed.ts` creates user `victor` as admin. Password is generated and stored in Bitwarden under the entry `BetelApp Admin — victor`. The seed is idempotent (uses `upsert`).
